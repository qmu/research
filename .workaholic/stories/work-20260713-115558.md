# Branch story — work-20260713-115558

## 1. Overview

This branch adds a seventh published research topic (image generation), makes every published article lead with a concise results overview, and restructures the whole publishing model into a dated survey-article series in which each topic's current page carries its own trend chart and links to all past surveys. The trigger was the owner spotting that the English "LLM response speed" page and the Japanese 応答速度 page had different structures; the root cause — one topic (speed) half-migrated to a separate "snapshot" surface whose Japanese translation was drawn from a different file than the English sidebar page — is fixed here by unifying all seven topics on one composed article model in which the Japanese page is always a translation of the English current page.

**Highlights:**

1. Image-generation benchmark: a new published topic comparing API-accessible image models (OpenAI gpt-image-1.5, Google Gemini 2.5 Flash Image, xAI Grok Imagine) by generation latency, per-image cost, rubric-checked prompt adherence, and exact-text rendering, judged mechanically by a fixed vision model
2. Site-wide §4 policy: every published article's Verification Results section is now a concise best/median/worst overview; the exhaustive per-subject tables moved to §7 Verification Data, machine-checked by a §4 compactness budget
3. Dated survey-article series (ADR 0006, supersedes 0005): each topic's current page is the latest survey's 7-section article composed with a 推移 (trend) block in §4 and a 過去の調査 (past surveys) links block in §7; earlier surveys stay as their own dated articles
4. English↔Japanese consistency restored: the real-run pipeline composes the current page and then translates it, so the two languages can never fork again (the reported speed bug)
5. Publish all runs to qmu: every dated Japanese survey is mirrored to qmu-co-jp so the in-article past-survey links resolve there

## 2. Motivation

The research site is the public face of qmu.co.jp's foundational LLM work, read by both people and AI agents. Two problems motivated this branch. First, coverage: clients increasingly ask about image generation, which the site did not measure. Second, structural integrity: the owner found the English and Japanese pages of the same topic structurally inconsistent, and — reviewing further — that the per-model result tables buried each article's takeaway and that the "snapshot" concept (a compact sidebar page separate from the full report) was both misleading in name and only ever half-built, for one topic. The fix had to restore a single, uniform article shape across all topics and both languages, driven from shared metadata so the two languages cannot drift, while giving each article the reader-first overview and the living-research trend-and-history the guideline promised.

## 3. Changes

The branch runs in three arcs: the new image-generation topic, the site-wide §4 overview policy, and the dated survey-article restructure with its publishing and documentation follow-through.

### 3-1. Add the image-generation benchmark research topic ([9d20583](https://github.com/qmu/research/commit/9d20583))

Added the seventh published topic under the proposal-first protocol (owner-approved design, fixture-only for this drive): a curated 3-model registry with web-verified ids and prices (Anthropic recorded as an honest non-subject), a versioned rubric prompt manifest, mechanical adherence/text scoring behind a fixed claude-sonnet-5 vision judge, an `ImageGenerationClient` port with OpenAI/Google/xAI/fixture adapters, unified-CLI wiring with fixture/estimate/real modes, and committed EN/JP pages passing every published-page guard. Estimated real trial cost ≈$0.95 against a $20 ceiling.

### 3-2. Apply the concise section-4 policy to every published article ([b11e2e4](https://github.com/qmu/research/commit/b11e2e4))

Generalized the speed/accuracy §4/§7 restructure (from the prior branch's editorial ticket) into a site-wide policy: OCR, RAG, availability, and the foundation-model catalog now open §4 with an aggregated overview (best/median/worst per metric, per-provider summaries; charts kept), with the raw per-subject tables moved intact to §7. A §4 compactness budget (3,000 characters, SVG excluded) machine-checks the policy over the committed pages, and the four Japanese pages were re-translated.

### 3-3. Compose each topic's current page as a dated survey-article ([6995212](https://github.com/qmu/research/commit/6995212))

Built the dated survey-article model: a pure composer (`current-article.ts`) injects a 推移 trend block into §4 and a 過去の調査 past-survey links block into §7 as bold-labelled blocks that keep the enforced 7-section outline intact. speed was unified off its snapshot special-case (its orphaned `.report.md` removed, its title aligned to its sidebar label), so its Japanese page is now a translation of its English current page — fixing the reported EN/JP inconsistency. A trend needs two distinct dates or it renders a first-survey note rather than a degenerate single-column chart.

### 3-4. Compose then translate the current page in the real-run pipeline ([f3cc3ac](https://github.com/qmu/research/commit/f3cc3ac))

Reconciled the `research -- <topic> --real` pipeline so it composes the freshly-rendered current page and then translates it (via the extracted `current-article-runner.ts`, shared with the `compose-current-articles` CLI command), instead of translating the insights prose. This means English → translate → Japanese holds automatically on every future real run, closing the class of bug that produced the speed inconsistency.

### 3-5. Publish every dated survey's Japanese article to qmu-co-jp ([5a8f87d](https://github.com/qmu/research/commit/5a8f87d))

Added `framePublishPlan` and extended the copy-plan / qmu-ticket commands and `publish-research.sh` so every dated Japanese survey is mirrored to qmu-co-jp under a path that equals the in-article link target — the 過去の調査 links resolve on qmu without any link rewriting. Verified end to end with copy-plan (11 frames) and a dry-run copy.

### 3-6. Supersede the snapshot ADR with the dated survey-article series ([c1a1cc2](https://github.com/qmu/research/commit/c1a1cc2))

Wrote ADR 0006 recording the dated-survey-article decision, marked ADR 0005 superseded, and restated the research development guideline and the living-research mission body in current-article / 推移 / 過去の調査 terms — removing the "snapshot" framing the owner found misleading and the now-defunct 1,500-token snapshot budget.

### 3-7. Add trend point extractors for OCR, image-generation, and RAG ([514d2a6](https://github.com/qmu/research/commit/514d2a6))

Added `statsRunsSnapshotPoints` (the `runs[].stats.<metric>.mean` shape used by OCR and image-generation) and `ragSnapshotPoints` (RAG's retrieval/operational shape), registered for those topics so their 推移 blocks chart real metrics once two same-instrument surveys exist. Availability keeps its native 90-day uptime chart by design.

## 4. Outcome

- **Seven published topics**: catalog, speed, accuracy, availability, OCR, RAG, and the new image-generation benchmark — each a dated survey-article series with a uniform shape in both languages
- **Reader-first articles**: every §4 is a concise best/median/worst overview; full evidence is one section down in §7; a machine-checked budget keeps §4 from regressing into a table dump
- **One article model, both languages**: the current page is the latest survey's 7-section article plus trend and past-survey blocks; the Japanese current page is a translation of the English current page, enforced by the pipeline
- **Living-research trend + history**: each article links every past survey and charts its metrics across the survey series (once two same-instrument surveys exist); past surveys stay as their own dated articles, never rewritten
- **Publishing**: every dated Japanese survey is mirrored to qmu-co-jp so past-survey links resolve there
- **Quality gates**: 304 vitest tests + tsc, `make lint` (0 warnings), `make build` (VitePress dead-link check), and all keyless fixtures green; the image-generation fixture is byte-stable across two runs; composition is byte-identical through the shared runner

## 5. Historical Analysis

The branch is the corrective completion of the living-research structure that ADR 0005 started. ADR 0005's two-surface "snapshot" split was applied to only one topic and forked its two languages by translating from a different file than it published — exactly the half-built structure that the prior branch's editorial work (title==label, no-mermaid guards) was hardening the repository against. This branch generalizes the shared-metadata principle one level further: not just order and labels but the entire article shape and its cross-survey composition are derived once and reused by both the CLI and the real-run pipeline, so the CLI and pipeline cannot drift and the two languages cannot fork. The §4 overview policy is the same guard-test-over-committed-files technique reused for a third invariant. Two of PR #15's deferred concerns are resolved here: "Only llm-speed is migrated to the snapshot structure" (all seven topics now share the model) and "JP pages are overwritten by the insights-translation stage after real runs" (the pipeline now composes-then-translates the current page, not the insights prose).

## 6. Concerns

### qmu English-section past-survey links resolve only in Japanese

- **Severity:** moderate
- **Description:** The past-survey block lists English + Japanese + data.json links per frame; D1 mirrors only the Japanese frames to qmu-co-jp, so on qmu the English/data links would be dead. research.qmu.dev resolves all links today (it hosts every frame), so this bites only at the next qmu `/ship` ([5a8f87d](https://github.com/qmu/research/commit/5a8f87d), see ticket 20260713144951).
- **How to Fix:** Make the block a single link per frame in the current page's own language (generated per language after translation, since translation keeps English URLs) and mirror the English frames into qmu's `docs/en/...history/` section; this needs a full re-translation of the current pages, best done as a focused follow-up before the qmu ship.

### Image-generation real adapters are not yet live-verified

- **Severity:** moderate
- **Description:** The OpenAI/Google/xAI image adapters and the vision-judge rubric are written and fixture-tested but never run against the real APIs (this drive was fixture-only by owner choice); the model ids and per-image prices were web-verified on 2026-07-13 but not exercised ([9d20583](https://github.com/qmu/research/commit/9d20583) in packages/tech/src/image-generation/)
- **How to Fix:** Run the first real trial (`npm run imagegen:estimate` then `imagegen:real`, ≈$0.95, $20 ceiling) to validate the adapters and rubric and to re-verify the ids at that moment; archive the dated frame

### Committed OCR/RAG/availability fixture pages summarize zero measured rows

- **Severity:** low
- **Description:** The committed current pages for topics whose fixture path produces no measured rows (OCR, RAG) show a "0 measured of N" overview and a first-survey trend note; they prove the pipeline but carry no comparison until a real run + archive ([b11e2e4](https://github.com/qmu/research/commit/b11e2e4), [6995212](https://github.com/qmu/research/commit/6995212))
- **How to Fix:** Run each topic's owner-gated real path and archive a dated survey; the overview and trend then populate from measured data

### The insights pipeline stage is now vestigial

- **Severity:** low
- **Description:** `.insights.md` is read by nothing now that the published Japanese page is a translation of the composed current page; the `insights` stage still runs on real mode and writes an unused file plus one LLM call per topic (ticket 20260713144951)
- **How to Fix:** Drop the `insights` stage from `topic.ts`; deferred because it cascades into stage-test churn, dead code, and stale committed `.insights.md` orphans for a negligible (monthly, owner-gated) saving

## 7. Successful Development Patterns

- **One composition path for CLI and pipeline:** extracting the compose logic into `current-article-runner.ts` and calling it from both the `compose-current-articles` command and the real-run pipeline means the two cannot drift — the exact failure (a second code path translating a different file) that caused the reported bug ([f3cc3ac](https://github.com/qmu/research/commit/f3cc3ac))
- **Derive the mirror destination to equal the link target:** D1 copies each Japanese frame to the same relative path the article already links, so publishing needs no link rewriting and the links resolve unchanged ([5a8f87d](https://github.com/qmu/research/commit/5a8f87d))
- **A real trend needs two distinct dates, not two points:** gating the trend chart on distinct dates (and showing a note otherwise) avoids a degenerate single-column chart when the current run and its own archived frame share a date ([6995212](https://github.com/qmu/research/commit/6995212))
- **Supersede ADRs, don't rewrite them:** ADR 0006 records the corrected decision and marks 0005 superseded, keeping the historical record intact while removing the misleading term from the living docs ([c1a1cc2](https://github.com/qmu/research/commit/c1a1cc2))
