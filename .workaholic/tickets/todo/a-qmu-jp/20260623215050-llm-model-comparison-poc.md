# Fundamental LLM model-comparison PoC and report

## Overview

Add the second foundational-research topic to `packages/tech`: a **fundamental
LLM model comparison** that scores multiple models/providers across eight aspects —
Provider, Model Name, Released, Cost, Effort Level, Speed, nested-JSON depth, and
length-specification accuracy. The first five are **curated reference data** (a
cited in-code registry); the last three are **measured live** against the real
provider APIs. This is the deliberate "minimum for now" — additional aspects
(more providers, more probes, effort sweeps) slot into the same shape later.

The topic mirrors the seed `llm-benchmark` layering verbatim — pure `domain/`
logic with unit tests, provider access behind the `vendors/llm/` anti-corruption
layer, a thin `entrypoints/` runner, a credential-free **fixture** path so CI
regenerates the report without keys, and a Markdown result page under
`docs/research-reports/` — widened to **three providers** (Anthropic + OpenAI +
Google) behind one `CompletionClient` contract. It also records the now-twice-
proven research-topic pattern as ADR 0004.

Decisions already settled with the developer: multi-provider now (one model each
for the minimum); curated-registry + 3-live-probe split; credentials via `.env`
(`ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `GOOGLE_API_KEY`). Approved plan:
`/home/ec2-user/.claude/plans/jazzy-fluttering-gray.md`.

## Implementation Steps

1. **Scaffold the topic.** Create `packages/tech/src/llm-model-comparison/` with a `domain/` subfolder, following `TEMPLATE.md` and the seed's layout.
2. **Domain types** (`domain/types.ts`): `Provider` union; `ModelCard` (static: `id`, `provider`, `modelName`, `apiModelId`, `released`, `inputCostPerMTok`, `outputCostPerMTok`, `effortLevels`, `source`); `Measurement` (live: `tokensPerSecond`, `maxNestedJsonDepth`, `lengthAccuracy`, `elapsedMs`, `outputTokens`, plus a `measured` boolean / fixture flag); `ComparisonRow = ModelCard & { measurement }`; `ComparisonResult`.
3. **Domain probes (pure + tested):**
   - `domain/nested-json.ts` — `buildNestedJsonPrompt(targetDepth)`, `jsonDepth(value)`, `gradeNestedJson(target, text) -> { parsed, achievedDepth, success }`. Runner sweeps a fixed depth ladder and records the deepest valid, correctly-nested response.
   - `domain/length-accuracy.ts` — `buildLengthPrompt(targetWords, topic)`, `wordCount(text)`, `lengthAccuracy(target, actual)` → `0..1`.
   - `domain/speed.ts` — `tokensPerSecond(outputTokens, elapsedMs)`.
   - `domain/report.ts` — `renderComparisonReport(result)` → Markdown with YAML frontmatter (incl. `description`), Method mermaid, the 8-column table, a clear curated-vs-measured legend, per-probe detail, and a Reproduce section. Reuse `llm-benchmark/domain/report.ts` escaping/frontmatter helpers.
   - Co-located `*.test.ts` for each, boundary-focused.
4. **Model registry** (`models.ts`): curated `ModelCard[]`, one model per provider, each with a cited `source`. Verify current Anthropic/OpenAI/Google model IDs, pricing, and release dates at implementation time (WebFetch provider pricing/model pages); isolate `apiModelId`s here.
5. **Extend the vendor ACL** (`vendors/llm/`): add `Completion` + `CompletionClient` to `types.ts`; add `createAnthropicCompletionClient` to `anthropic.ts` (capture `usage.output_tokens` + wall-clock; no thinking param, final-answer-only instruction, `max_tokens` ~2048); add new `openai.ts` and `google.ts` adapters using the official `openai` and `@google/genai` SDKs; add `createFixtureCompletionClient` to `fixture.ts` returning deterministic `Completion`s keyed by prompt shape. Do not touch the seed's `LlmClient`.
6. **Entrypoint** (`entrypoints/run-llm-model-comparison.ts`): build each model's client by provider; with `--fixture` or a missing provider key, use the fixture client and flag that row as fixtured (never present it as measured); run the probes, derive speed, assemble rows, render to `docs/research-reports/llm-model-comparison.md` (`OUTPUT_PATH` overridable), print a summary table. Extend `src/index.ts` usage text.
7. **Wire ops/docs/CI:** add `compare` / `compare:fixture` scripts (load `.env` via Node `--env-file-if-exists=.env`; no `dotenv` dep needed — Node is v24) + `openai` / `@google/genai` deps to `package.json`; record the new SDKs in `docs/dependency-decisions.md`; populate the VitePress sidebar `items` and `docs/research-reports/index.md` with both reports; add a `compare:fixture` step to `.github/workflows/build-research-tech.yml`; add `.env.example`.
8. **Record the pattern:** write `docs/adr/0004-research-topic-anatomy.md` capturing the now-twice-proven anatomy (pure `domain/` + tested, per-provider `vendors/` ACL, thin `entrypoints/` runner, curated-registry-vs-live-measurement split, fixture path for CI, report → publish).
9. **Verify:** `npm install` in `packages/tech`; `make test` (type-check + vitest); `make lint`; `npm run compare:fixture` (regenerate report with no keys; inspect 8-column table + legend); `make build` + `make a11y`. **Pause for the developer to populate `.env`**, then `npm run compare` for the real multi-provider run; confirm live numbers and that any key-less provider is flagged fixtured.

## Considerations

- **Do not weaken the seed contract.** Add `CompletionClient` alongside the existing `LlmClient`; the seed's `generateAnswer` and its tests must keep passing (`packages/tech/src/vendors/llm/types.ts`).
- **Honesty of the report is a policy requirement.** A provider without a key in a real run is fixtured-and-flagged, never silently presented as a live measurement (`domain/report.ts`, `entrypoints/run-llm-model-comparison.ts`) — `policies/objective-documentation.md`.
- **Vendor neutrality across three SDKs.** Keep `openai` / `@google/genai` types strictly inside `vendors/llm/{openai,google}.ts`; the domain depends only on `CompletionClient`/`Completion` — `policies/vendor-neutrality.md`. Record both deps in `docs/dependency-decisions.md`.
- **Model-ID accuracy.** OpenAI/Google model IDs, pricing, and release dates are uncertain across the knowledge cutoff — verify against live provider docs when filling `models.ts`; keep IDs isolated there for easy correction.
- **CI must stay keyless.** The added workflow step runs `compare:fixture` only; the real `compare` target is never run in CI (`.github/workflows/build-research-tech.yml`) — `policies/ci-cd.md`, `policies/command-scripts.md`.
- **Accessibility of the generated table.** The 8-column table and legend must keep `make a11y` green (semantic headers, stable anchors) — `policies/accessibility-first.md` (`docs/.vitepress/`).
- **Effort/probe scope is intentionally minimal.** Effort Level is curated metadata for now, not a measured sweep; "another aspect" (effort sweeps, more providers/probes) is explicitly deferred to a follow-up.
