# Glossary

One concept, one word. Search this list before introducing a new term, and use
the same word across code, documents, commits, diagrams, and conversation.

| Term              | Meaning                                                                 |
| ----------------- | ----------------------------------------------------------------------- |
| research          | A self-contained, reproducible study shipped as code plus a result page. |
| topic             | One research, living as a subfolder under a package's `src/`.            |
| tech              | The technical research group (`packages/tech/`): benchmarks, architecture studies. |
| industry          | The industry/market research group (`packages/industry/`).              |
| benchmark         | A research that measures and compares the behavior of systems or models. |
| result page       | The Markdown write-up of a research, under `docs/research-reports/`.     |
| preview site      | The local VitePress site that renders result pages before publication.  |
| publish           | Copy a finished result page to the corporate site (`../qmu-co-jp`).      |
| foundational research | Research published on qmu.co.jp under the `/research` section.       |
| trial             | One full execution of a topic's measurement, producing a data artifact and report at a point in time. |
| uniform trial report | The dated frame a trial leaves under `docs/research-reports/history/`, in the same format every time. |
| cadence           | The agreed interval between recurring trials of a topic.                 |
| tendency window   | The span of recent trials a snapshot summarizes (the last 3–5 months).   |
| snapshot          | A topic's sidebar page: the compact latest view of tendencies over the tendency window, linking to each uniform trial report. |

Bring proprietary terminology discovered during industry research back into this
glossary so it becomes the shared vocabulary.
