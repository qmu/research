# SVG生成

このレポートは、フロンティアのテキストモデルが**妥当でアニメーション可能なベクターグラフィックス**をどれだけ忠実に生成できるかを比較します。ソースレベルのスコアは**SVGソースから機械的に**算出されます——整形式判定、描画要素とパスコマンドの数、アニメーション記述の有無。**プロンプト忠実度**は各描画をラスタライズし、固定のビジョン判定モデルがバージョン管理されたYes/Noルーブリックに回答します。すべての制約は機械的に検証可能であり、美的な主観評価は数値に一切含まれません。

## 1. 調査の目的

本調査の目的は、どのフロンティアモデルが実際にパース可能なSVGを生成するか、描画が指示どおりの内容か（プロンプト忠実度）、どれだけの描画量を持つか、モーション（SMILまたはCSSアニメーション）を表現できるか、どれほど高速に応答が返り、どれだけのトークンコストがかかるか——つまり、あるモデルが製品内でベクターグラフィックス生成を担えるかを左右する特性を記録することである。

## 2. 測定対象

### 対象モデル

対象は、選定済みレジストリ（`packages/tech/src/svg-generation/models.ts`）に含まれる4つのテキストフラッグシップであり、プロバイダーごとに1モデルが、引用元および最終確認日と共に掲載されている。SVGは各プロバイダーの通常の補完（completion）APIを通じて出力されるため、専用の画像エンドポイントは存在せず、対象外となるプロバイダーもない。

### 対象メトリクス

測定対象のメトリクスは、描画妥当性（`<svg>`をルートとする整形式XMLである割合、高いほど良い）、プロンプト忠実度（満たしたルーブリック制約数／総数。`fixture-raster`でラスタライズした描画を固定の`fixture-judge`ビジョン判定器が採点、高いほど良い）、パス複雑度（描画要素＋パスコマンド数、記述的指標）、アニメーションの有無（アニメーション用プロンプトのうちSMIL/CSSアニメーションを含む割合、高いほど良い）、生成レイテンシ（ms、低いほど良い）である。トークンコストは実測の出力トークン数×カタログ価格から導出される（参考情報）。

## 3. 範囲と制約

- **機械的評価であり、美的評価ではない。** ソースレベルのスコアはSVGソースが明かす情報のみを読み取る。プロンプト忠実度は描画をラスタライズし、固定のビジョン判定器がバージョン管理されたYes/Noルーブリック制約に回答する——チェックリストであり、好みの採点ではない。
- **忠実度の計測器は固定かつバージョン管理されている。** 判定モデル（`fixture-judge`）、ラスタライザエンジン（`fixture-raster`）、ルーブリックはすべてマニフェストバージョン`svg-v2`に属する。いずれかの差し替えはバージョンアップであり、暗黙の変更は行わない。判定器は静止フレームを読むため、忠実度は「何が描かれたか」を採点し、モーションはソースレベルのアニメーション有無メトリクスが担う。
- **レンダリング不能なSVGの忠実度は0**（判定器の読み取りなし）。描画妥当性は依存関係のない構造的パースのままであり、キー不要経路はネイティブのラスタライザを必要としない。
- プロンプトマニフェストのバージョンは`svg-v2`：5個のプロンプト（静的3個、アニメーション2個）、各プロンプトに忠実度ルーブリック付き。履歴は同一マニフェストバージョンの地点同士のみを接続する。
- フィクスチャ経路はキー不要かつ決定論的である（フィクスチャ生成器・フィクスチャラスタライザ・フィクスチャ判定器）。実際のモデルの数値は、オーナーが承認済みのコスト上限内で実経路を実行した場合にのみ現れる（まず`--estimate`を実行すること）。
- 特定時点のもの：計測された挙動は`2026-01-01T00:00:00.000Z`時点のモデルおよびAPIを反映している。カタログ価格は各行の最終確認日時点のものである。

## 6. 再現方法

### 再現手順

```sh
git clone https://github.com/qmu/research
cd research/packages/tech
npm install

# キー不要のセルフテスト（決定論的なフィクスチャクライアント）:
npm run research -- svg-generation --fixture

# コストの事前確認、その後オーナー承認による実行:
npm run research -- svg-generation --estimate
npm run research -- svg-generation --real
```

### 再現コスト（目安）

フィクスチャ経路はキー不要かつ無償である。実トライアルは各プロバイダーに生成トークン分（SVG1枚あたり数百出力トークン程度）が課金され、加えて生成されたSVG1枚につき固定ビジョン判定器の読み取りが1回発生する（ラスタライズ自体はローカルで無償）。合意された上限は1トライアルあたり$5であり、まず`--estimate`を実行しなければならない。

### クリーンアップ

外部リソースは一切作成されない。SVGはメモリ内で生成・採点され、実行が書き出すのはローカルのMarkdown/JSONアーティファクトのみである——コミット前に内容を確認すること。
