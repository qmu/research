# OCR能力

本レポートは、LLM基盤調査のためのOCR比較ハーネスを記録するものです。コミットされたレポートは、鍵不要（keyless）のフィクスチャパスによって生成されています。

## 1. 調査の目的

決定論的なドキュメント画像に対してOCRを実行し、転記テキストと構造化フィールドの両方を返すことができる基盤モデル統合はどれかを検証することが目的である。

## 2. 測定対象

### 対象モデル

対象行は OCR ランナーにおけるモデル候補である。実際のモデル数値は、行の provenance が `measured` である場合にのみ存在する。`out-of-scope` の行は、登録された `VisionClient` アダプタを持たないモデルを示す。

### 対象メトリクス

測定対象のメトリクスは、文字誤り率（CER）、単語誤り率（WER）、およびフィールドレベルの構造化抽出精度である。必須フィールドは `document_type`、`vendor`、`document_id`、`date`、`currency`、`total` である。

## 3. 範囲と制約

- フィクスチャの行はハーネスチェックにすぎない。これらは、文書レンダリング、ビジョンポートの配線、CER/WER、フィールドスコアリング、集計、レポートレンダリングが決定論的であることを証明するものである。
- `out-of-scope`（対象外）と記された行は、画像パスを通じて送信されない。プロバイダー／モデルは、OCR実行に組み込まれる前に、明示的な`VisionClient`アダプターを必要とする。
- 実モデルの数値は、オーナーが`--estimate`を確認した上で実パスを実行するまでは、未計測のままである。
- 合成フィクスチャは、クリーンなラテン文字／数字の文書レイアウトのみを対象としている。日本語OCR、手書き文字、傾き、スタンプ、スキャン、写真、複数ページ分割は測定していない。

## 4. 検証結果

今回の実行では、19個のモデル行のうち**0件が測定済み**です（未測定の行は`fixtured`のハーネスチェック、または`out-of-scope`の統合ギャップです）。

集計すべき測定値は存在しませんが、コミットされたフィクスチャページによってハーネスがエンドツーエンドで機能することが証明されています。モデルごとの表はセクション7、検証データに記載されています。

これはこのシリーズにおける最初の比較可能な調査であるため、まだ複数調査にわたるトレンドを図示することはできません。トレンドチャートは、同一計測手法による2回目の調査がアーカイブされた時点で表示されるようになります。それまでの調査は検証データの下にリンクされています。

## 5. 考察

`measured` の provenance を持つ行は、OCR品質について比較可能である。`out-of-scope` の provenance を持つ行は、画像パスを一度も通っていないため、OCRの失敗ではなく、統合カバレッジの欠落を示すものである。

## 6. 再現方法

### 再現手順

```sh
cd packages/tech
npm install

npm run ocr:fixture
npm run ocr:estimate
npm run ocr:real
```

### 再現コスト（目安）

`ocr:fixture` はキー不要かつコストがかかりません。`ocr:estimate` はオーナー限定の実データ経路を事前確認するものです。`ocr:real` は設定済みのビジョン対応プロバイダーを呼び出し、プロバイダー側のトークン/画像料金が発生します。

### クリーンアップ

OCRのフィクスチャ経路および実データ経路は、外部クラウドリソースを作成しません。これらはローカルのレポート/データ成果物を生成するため、コミット前にそれらのファイルを確認してください。

**データセット。** QMU合成ドキュメントOCRフィクスチャ v1（`qmu-synthetic-document-ocr-v1`）、マニフェストバージョン 2026-07-08。出典：リポジトリが作成した決定論的な合成ドキュメントマニフェスト。ライセンス：MIT。生成されたドキュメントテキスト、フィールドの正解データ、およびレンダリングされた画像は本リポジトリのために作成されたものであり、リポジトリのライセンスの下で再配布可能です。画像はこのマニフェストから実行時に決定論的にレンダリングされ、サードパーティのドキュメント画像はコミットされていません。

**前処理。** 画像は image/png として 960×640px、150 DPI でレンダリングされます。1つのマニフェスト項目が1枚の画像／ページとしてレンダリングされ、複数ページのドキュメントはこのv1フィクスチャの対象外です。白背景、黒の固定 5x7 ビットマップテキスト、回転なし、ぼかしなし、圧縮による劣化なし。

**言語とレイアウト。** ラテン大文字、アラビア数字、ASCII記号、ISO日付形式、および通貨コード。日本語の文字はこのフィクスチャには含まれていません。見出し、整列されたラベル、表形式の項目行、合計フィールドを持つ、レシート／フォーム複合の単一ページ形式です。手書き文字、スタンプ、写真、傾き、重なり合う列は対象外です。

**採点方法。** CERおよびWERは編集距離を参照テキストの長さで割ったものです。フィールド精度は、正規化されたフィールドの完全一致数を必須の正解フィールド数で割ったものです。正規化：採点前に連続する空白を1つのASCIIスペースに圧縮し、両端をトリムし、改行コードを正規化します。CER／WERでは句読点を保持します。フィールド値については、数値合計から桁区切りのカンマを除去し、通貨コードを大文字化します。
