# OCR能力

このレポートは、LLM基盤研究のためのOCR比較ハーネスを記録したものです。コミットされたレポートは、キー不要のフィクスチャパスによって生成されています。

## 1. 調査の目的

決定論的なドキュメント画像に対してOCRを実行し、書き起こしテキストと構造化フィールドの両方を返せる基盤モデル統合はどれかを検証することが目的である。

## 2. 測定対象

### 対象モデル

対象となる行は、OCRランナーにおけるモデル候補です。実際のモデル数値は、行のprovenanceが`measured`である場合にのみ存在します。`out-of-scope`の行は、登録された`VisionClient`アダプターを持たないモデルを示しています。

### 対象メトリクス

測定対象のメトリクスは、文字誤り率（CER）、単語誤り率（WER）、およびフィールドレベルの構造化抽出精度です。必須フィールド：`document_type`、`vendor`、`document_id`、`date`、`currency`、`total`。

## 3. 範囲と制約

- フィクスチャの行はハーネスチェックにすぎない。文書レンダリング、ビジョンポート配線、CER/WER、フィールドスコアリング、集計、レポートレンダリングが決定的であることを証明するものである。
- `out-of-scope` と記された行は画像パスを経由して送信されていない。プロバイダ／モデルがOCR実行に組み込まれるには、明示的な `VisionClient` アダプタが必要である。
- 実モデルの数値は、オーナーが `--estimate` を確認した上で実パスを実行するまで、未計測のままである。
- 合成フィクスチャは、クリーンなラテン文字／数字の文書レイアウトのみを対象としている。日本語OCR、手書き文字、傾き、スタンプ、スキャン、写真、複数ページ分割は計測対象外である。

## 6. 再現方法

### 再現手順

```sh
cd packages/tech
npm install

npm run ocr:fixture
npm run ocr:estimate
npm run ocr:real
```

### 再現コスト（目安）

`ocr:fixture` はキー不要でコストもかかりません。`ocr:estimate` はオーナー限定の実実行パスをプレビューします。`ocr:real` は設定済みのビジョン対応プロバイダーを呼び出し、プロバイダー側のトークン/画像料金が発生します。

### クリーンアップ

OCRのフィクスチャおよび実実行パスは、外部クラウドリソースを作成しません。ローカルのレポート/データ成果物を生成するため、コミット前にそれらのファイルを確認してください。

**採点。** CERおよびWERは編集距離を参照テキストの長さで割った値です。フィールド精度は、正規化後のフィールドが完全一致した数を、必須の正解データフィールド数で割った値です。正規化：採点前に、連続する空白を1つのASCIIスペースに圧縮し、両端をトリムし、改行コードを正規化します。CER/WERでは句読点を保持します。フィールド値では数値の合計からグループ区切りのカンマを除去し、通貨コードを大文字にします。
