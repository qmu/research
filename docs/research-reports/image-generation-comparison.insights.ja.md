# 画像生成

このレポートは、画像生成モデルを**機械的に検証可能な**挙動のみで比較します——固定された画像判定モデルが、画像ごとに決定論的なyes/noのルーブリックに回答する方式であり、美的な主観評価はスコアに一切含まれません。

## 1. 調査の目的

本調査の目的は、APIでアクセス可能な画像生成モデルにはどのようなものが存在し、1枚あたりの画像生成コストがいくらで、どれほど高速に応答が返り、そして検証可能なプロンプト制約にどの程度忠実に従い、正確なテキストをどれだけ精密にレンダリングできるか——つまり、統合方式の選定を左右する特性を記録することである。

## 2. 測定対象

### 対象モデル

対象は、選定済みレジストリ（`packages/tech/src/image-generation/models.ts`）に含まれる3つの画像生成モデルであり、対象となる各プロバイダーにつき1モデルが、引用元および最終確認日と共に掲載されている。

- **Anthropic** は対象外である：画像生成APIを提供していないため（2026-07-13時点で確認済み）。

### 対象メトリクス

測定対象のメトリクスは、生成レイテンシ（ms、低いほど良い）、プロンプト遵守度（満たされたルーブリック制約数／全体数、高いほど良い）、テキストレンダリング精度（ビジョン文字認識で検出された期待トークン数／期待トークン数、高いほど良い）である。1画像あたりのコストは選定済みカタログデータ（参考情報）であり、実測値ではない。

## 3. 範囲と制約

- **判定はするが、評価基準に制約がある。** 固定されたビジョン判定器（`fixture-judge`）が決定論的なyes/no質問に回答し、レンダリングされたテキストを書き起こす。美しさやスタイルを採点することは一切ない。判定器の差し替えは通常の更新ではなく、計測器の変更に相当する。
- プロンプトマニフェストのバージョンは`1`：8個のプロンプト（6個のルーブリック、2個の完全一致テキスト）。履歴は同一マニフェストバージョンの地点同士のみを接続する。
- **画像バイナリはコミットされない。** アーティファクトにはバイト長、タイミング、判定器の回答、スコアが記録され、このページを再生成するのに十分な情報が含まれるが、画像そのものは含まれない。
- フィクスチャ経路はキー不要かつ決定論的である。実際のモデルの数値は、オーナーが承認済みのコスト上限内で実経路を実行した場合にのみ現れる（まず`--estimate`を実行すること）。
- 特定時点のもの：計測された挙動は`2026-01-01T00:00:00.000Z`時点のモデルおよびAPIを反映している。カタログ価格は各行の最終確認日時点のものである。

## 6. 再現方法

### 再現手順

```sh
git clone https://github.com/qmu/research
cd research/packages/tech
npm install

# キー不要のセルフテスト（決定論的なフィクスチャクライアント）:
npm run research -- image-generation --fixture

# コストプレビュー、その後オーナー限定の実実行:
npm run research -- image-generation --estimate
npm run research -- image-generation --real
```

### 再現コスト（目安）

フィクスチャ経路はキー不要でコストもかからない。実トライアルでは、生成画像ごとに各プロバイダーへの課金が発生し（モデルごとのカタログ価格を参照）、加えて画像ごとに1回のビジョンジャッジ読み取りが発生する。合意された上限は1トライアルあたり$20であり、`--estimate` を先に実行する必要がある。

### クリーンアップ

外部リソースは作成されない。生成された画像は判定のためにメモリ上に保持され、その後破棄される。実行時にはローカルのMarkdown/JSON成果物のみが書き出されるため、コミット前にそれらを確認すること。
