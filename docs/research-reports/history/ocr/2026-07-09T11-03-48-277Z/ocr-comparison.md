# OCR comparison

This report records the OCR comparison harness for LLM foundation research. The committed report is generated by the keyless fixture path. Real model numbers are not present unless provenance is `measured`.

## Methodology

**Dataset.** QMU synthetic document OCR fixture v1 (`qmu-synthetic-document-ocr-v1`), manifest version 2026-07-08. Source: Repository-authored deterministic synthetic document manifest. License: MIT; generated document text, field ground truth, and rendered images are authored for this repository and may be redistributed under the repository license. Images are rendered deterministically at run time from this manifest; no third-party document images are committed.

**Preprocessing.** Images are rendered as image/png at 960×640px, 150 DPI. One manifest item is rendered as one image/page; multi-page documents are out of scope for this v1 fixture. White background, black fixed 5x7 bitmap text, no rotation, no blur, no compression loss.

**Language and layout.** Latin uppercase letters, Arabic numerals, ASCII punctuation, ISO dates, and currency codes. Japanese script is not included in this fixture. Single-page receipt/form hybrid with headings, aligned labels, tabular item lines, and total fields. Handwriting, stamps, photos, skew, and overlapping columns are out of scope.

**Scoring.** CER and WER divide edit distance by the reference length. Field accuracy is exact normalized-field matches divided by required ground-truth field count. Normalization: collapse runs of whitespace to one ASCII space, trim ends, and normalize line endings before scoring. preserve punctuation for CER/WER; field values strip grouping commas from numeric totals and uppercase currency codes.

**Structured schema.** Required fields: `document_type`, `vendor`, `document_id`, `date`, `currency`, `total`.

## Scope and limitations

- Fixture rows are harness checks only. They prove that document rendering, vision-port wiring, CER/WER, field scoring, aggregation, and report rendering are deterministic.
- Rows marked `out-of-scope` are not sent through the image path. A provider/model needs an explicit `VisionClient` adapter before it can enter the OCR run.
- Real model numbers remain 未測定 until an owner runs the real path after reviewing `--estimate`.
- The synthetic fixture covers clean Latin/digit document layouts only. It does not measure Japanese OCR, handwriting, skew, stamps, scans, photos, or multi-page splitting.

## Reproduce

```sh
cd packages/tech
npm install

# Keyless byte-stable fixture:
npm run ocr:fixture

# Estimate before any real vision run:
npm run ocr:estimate

# Owner-gated real path:
npm run ocr:real
```
