## 5. 考察

これらの数値は公開ステータスページの観測記録として読むべきである。導出アップタイムが高い場合でも、対象期間中の自己申告インシデント記録に重み付き停止時間が少なかったことを示すだけで、実際のAPI可用性や将来の信頼性を証明するものではない。

## 6. 再現方法

### 再現手順

```sh
git clone https://github.com/qmu/research
cd research/packages/tech
npm install

npm run availability:fixture
npm run availability:estimate
npm run availability
```

### 再現コスト（目安）

`availability:fixture` はキー不要でコストなし。`availability:estimate` は取得・書き込みを行わずLLM抽出ステップを見積もる。`availability` は公開ステータスページを取得し、設定されたLLM抽出器を使うためプロバイダーモデルのトークンコストが発生する。

### クリーンアップ

フィクスチャパスは外部リソースを作成しない。実パスではローカルの蓄積履歴とレポートアーティファクトを更新するため、コミット前に差分を確認する。プロバイダー側リソースは作成しない。
